# Capítulo 27: Guia para Desenvolvedores

## Padrões de Código
- **Tailwind:** Utilize as classes utilitárias. Evite CSS personalizado (inline styles).
- **TypeScript:** Tipagem estrita em todos os componentes e serviços.
- **Componentes UI:** Utilize a base do Shadcn/UI e estenda conforme necessário.

## Organização de Módulos (Frontend)
- Lógica de negócio em `src/features/[module-name]`.
- Componentes de interface pura em `src/components/ui`.
